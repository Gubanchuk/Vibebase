# Три функции pushed output (канон)

### 1. Noticing function
Когда пытаешься сказать что-то на L2 и не можешь — замечаешь GAP между
intended meaning и available form. Это триггер к обучению. Без попытки
output'а этот gap невидим.

### 2. Hypothesis testing function
Каждый output — гипотеза «так говорят». Реакция собеседника (непонимание,
поправка, подтверждение) — данные для hypothesis testing.

### 3. Metalinguistic function
Output заставляет подумать о форме, не только о смысле. Коллаборативное
письмо, когда партнёры обсуждают «как правильно X», даёт сильное
metalinguistic осознание.

# Как применять в нашем тьюторе

- **Mode 1 (chat):** после 3-4 turn'ов request pushed output — «попробуй
  пересказать своими словами»
- **Mode 2 (lesson):** exercise-section должен требовать production, не
  только recognition
- **Mode 4 (quiz):** дополнить fill-in-blank-заданиями где юзер пишет
  слово, не выбирает из списка

# Баланс с Input Hypothesis

Krashen и Swain не противоречат — они дополняют. Input нужен (подаёт
материал), output нужен (форсит processing + feedback loop). Балансирoвка:
~70% input, ~30% output для intermediate; обратное для advanced.
